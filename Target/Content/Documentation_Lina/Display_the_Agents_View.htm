Display the Agents View

In the , select > Agents > Main View.

The agents view enables to manage the agent parameters (see Table).

Agent view description

Column

Description

Block status

Block status.

Tags

Tags used per agent.

Tenant

Tenant used for the agent.

Node

Node used for the agent (multi-server only)

Last logon

Allows the administrator to check the name of the last user logged in to a session on the agent.

Version

Current version number of the agent.

Agent Group

Name of the group to which the agents belong.

Status

Status of the agent. For instance, Active.

Host name

Displays as the agent identifier as declared on the server. Generally, the agent identifier is identical to the computer name. If the computer name already exists e.g. on a machine in a different domain, adds a suffix in the form -nnn (from 000 to 999). To use a dedicated agent identifier when creating a new agent, see Agent to Server Access Configuration.
The agent identifier can be modified by double clicking the agent in the Agent Details window.

Computer name

Name by which the agent is identified on the network. Frame background turns red when the computer raises an alarm.

Protection

Protection scheme currently applied to the .

Strategy

Strategy currently applied to the .

Protected volume

Indicates the protected volume.

Quotas

Indicates the percentage of quota used. See Creating a Data Protection Strategy for details on how to set quotas.

Replication source name

Name of the server being replicated on the current server.

If you are viewing the from the replication source server (or if no replication is defined), this field is empty.

Replication destination name

Name of the server to which the current server is replicated.

If you are viewing the from the replication destination server (or if no replication is defined), this field is empty.

Replication lag

Displays information regarding the replication of this agent. Synchronized means the agent has been correctly replicated to the target replication server.

Last synchronization point replicated

Time of the last synchronization, i.e., last protection time of the replicated data.

Last connection

Time at which the last contacted the server.

Last synchronization

Time at which the last data to be protected was sent to the server from the agent. The update of the Synchronization time consists of these steps:

Each time a protection session completes without any error i.e. all files and directories are protected, the Synchronization time is updated along with the Last session completed. If errors occur, for instance one file is missing, then the Last session completed time is updated, but not the Synchronization time.

If no files are modified at the time a session is supposed to occur (according to the RPO parameter) i.e. there is no new data to protect, the Synchronization time is updated nonetheless to indicate that data is protected. In that case, the session time (started and completed) are not updated.

E-mail

Indicates the e-mail address provided by the user on the agent. If this field is empty, the administrator has the ability to enter an e-mail address. This enables users to:

Connect to the Web restore application and receive the web restore code to restore data.

Receive notifications about the protection status of their agent.

Once the administrator fills in the e-mail on the server side, it can no longer be modified on the agent. See Assigning an Email Address to the Agent for details.

CDP type

For example OS File System Events or the Driver providing a way for to request notifications that allow to quickly detect and backup modified files.
