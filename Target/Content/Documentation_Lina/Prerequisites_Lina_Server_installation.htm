Prerequisites to Server Installation

Before installing a server:

Check the following prerequisites (see Table).

Refer to the Compatibility Guide to verify the list of supported operating systems for the server.

Prerequisites to server installation

Prerequisite

Description

Hardware/Processor

The server must be installed:

On a machine with a multi-core 64-bit processor.

On a local and dedicated fast access disk, such as an SSD.

Operating System

To find out the list of supported Operating Systems, see the Compatibility Guide.

Linux Server

Some packages must be installed before running the installer.

For an installation in CLI using the silent mode, run the following commands to install the required packages:

Debian-based distributions: apt-get update && apt-get install -y --no-install-recommends libaio1 libcrypt libavahi-client3 lsb-core

RPM-based distributions: yum update && yum install libaio libcrypt avahi-libs

For an installation in graphical mode, run the following commands to install the required packages:

Debian-based distributions: apt-get update && apt-get install -y --no-install-recommends libaio1 libcrypt libavahi-client3 lsb-core libxext6 libxtst6 libxrender1 fonts-dejavu

RPM-based distributions: yum update && yum install libaio libcrypt avahi-libs libXext libXtst libXrender dejavu-sans-fonts dejavu-serif-fonts

To use the graphical environment of a remote host during an installation in graphical mode, one of the following items must be used:

A graphical environment (X Windows) on the local host.

A DISPLAY variable set .

Linux Server Snapshot

If you intend to use snapshots to secure your Linux Server, all your ADE components must be stored on:

Version 2.02.95 or higher of LVM2

Thinly-Provisioned Logical Volumes (thin volumes)

For more details on how to create a thin volume, see Create a Thin Volume on Linux.

Windows Server Snapshot

If you intend to use snapshots to secure your Windows Server, store all your ADE components on an NTFS file system.

If you intend to use the snapshot revert feature, do not install any ADE components on the system disk.

File system

It is recommended to use:

An ext4 file system on Linux servers

An NTFS on Windows servers.

Memory

At least 8 GB of RAM is required.

Repository

At least 32 GB of disk space must be permanently available on the volume where the Atempo-Deduplication Engine data blocks are stored.

Directory of the server Database

At least 10 GB of disk space must be permanently available on the volume where the data is stored.

Default Communication Ports

These ports are the default ports and must be open on your firewall (unless you choose other values when you run the setup):

8181: Communication port for .

23232: Communication port for Atempo-Deduplication Engine.

Proxy option in web browser

Ensure that your web browser does not use a proxy. Otherwise, you may get this error message when trying to access your server in the : "Attempt to load data from a server outside the security sandbox".

To avoid this problem, follow these instructions:

Mozilla Firefox: Open the Application Menu > Settings > Network Settings. In the Connection Settings window, select the No Proxy check box.

Microsoft Internet Explorer: Open Tools > Internet Options > Connections > LAN Settings. In the Local Area Network (LAN) Settings window, ensure that the Use a proxy server for your LAN option is unchecked.

Google Chrome: Open the menu, then click Settings > Advanced > Open proxy settings. In the Proxy window, ensure that the Use a proxy server option is unchecked.

Apple Safari: Open Safari > Preferences > Advanced tab > Change Settings.... Ensure that no proxies are used.

Language

adopts the language set at the operating system level. Available languages are English and French.

User

The user launching the setup program must have administrator or root rights.

During the installation or upgrade, the setup verifies that prerequisites are met. If your environment does not meet some of them, you may encounter one of the following cases:

A warning message specifies which requirement is not met, as well as recommendations on the corrective measures to take to comply with the requirements.

The installation stops until you make the necessary changes to meet the prerequisites.
