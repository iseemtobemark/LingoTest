Prerequisites to Linux Agents Installation

Before installing  Linux Agents, check the following prerequisites (see Table):

Prerequisites to Linux agent installation

Prerequisite

Description

Operating System

To find out the list of supported Operating Systems, see the Compatibility Guide.

Memory

A minimum of 512 MB of RAM is required.

Database Directory

The database requires approximately 1% of the volume of the protected data in disk space, but can use up to 10% if you have a large number of files.

Setup Directory

The directory path where you launch the setup must not contain space characters as this is currently not supported.

Product Download

Ensure that you have downloaded the binaries from the Atempo Client Portal: https://support.atempo.com.

Communication Port

If you want to connect to the Agent interface remotely, for instance to perform remote restores, port 27261 must be open on your firewall.

Language

adopts the language set at the operating system level. Available languages are English and French. Ensure that you set the proper language for the OS.

Packages

Some packages may be required, before running the installer. For an installation on a non-graphical system, there is no requirement. For an installation on a graphical system, using a Gnome environment, run the following commands:

Debian-based distributions: apt-get update && apt-get install -y --no-install-recommends libxss1 python-nautilus gnome-shell-extension-top-icons-plus

RPM-based distributions: yum update && yum install libXScrnSaver nautilus-python gnome-shell-extension-top-icons

gnome-shell-extension-top-icons is required to enable icon in the Gnome task bar.

For an installation on an Ubuntu environment with a Unity graphical shell, run the following command: apt-get update && apt-get install -y --no-install-recommends libxss1 python-nautilus python-appindicator python-gtk2

python-nautilus (Debian) / nautilus-python (RPM) is required to enable context menus in Nautilus.

For an installation on a CentOS-based distribution, the EPEL repository may need to be enabled (Extra Packages for Enterprise Linux). To find all the required packages, use the following command: yum install epel-release.
