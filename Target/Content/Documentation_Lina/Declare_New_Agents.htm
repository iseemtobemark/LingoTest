Declare New Agents

Once the agents are installed, you must add them in the on the server. When an agent is added on the , you can define the agents default settings and configure a protection, by applying a protection scheme and a strategy.

You can add the agents in two ways:

By enabling the Agent auto-creation option. This option cannot be used in a multi-server configuration.

By declaring them manually in the list of agents.

To open the list of agents in the

To open the list of agents, go to Agents.

In the list of agents, you can:

Manage agents that are not declared. To declare an agent, click the button. Select the agent to declare, a protection, a strategy and an agent group, then click Create.

When declaring a macOS agent named by default "user’s Mac", we recommend to select the agent from the list rather than typing its name. If you do type it, note that the apostrophe in "user’s" is a typographic apostrophe (unicode 2019, UTF-8 E2 80 99) and not the apostrophe from the keyboard (0x27).

Configure the global agents parameters (only on a master or a standalone server)

Refresh the list of agents.

Export the list of tenants in CSV or HTML format.

Delete or undelete an agent, set the block status of an agent or move that agent to another tenant. All these actions can be performed when an agent is selected.

Display or hide specific information for all agents, such as the block status of an agent or the protected volume of an agent. This information appear in the list of agents. To hide or display a column, click on the ellipsis button on the right-hand of the window, in the column row.

Filter per column row, type of alert, tag, block status or advanced filters (number of agents to display on a page, the last time an agent synchronized with the server, for instance).
