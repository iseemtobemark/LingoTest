Set an Encryption Key for Data Blocks

You can encrypt data blocks in the Repository.

The encryption is:

Performed by the Server when writing blocks to the repository.

Based on the AES-256 algorithm.

Blocks that were already stored in the repository prior to the activation of the encryption are not encrypted. To encrypt them, you have to compact blocks: in this way, blocks are re-written and encrypted.

To set an encryption key

Go to Settings > Preferences > Encryption.

Enter the key of your choice or click on Generate a key.

Once you have defined an encryption key, it cannot be changed.

Click Activate encryption.

The key is used for all agents that are protected by the Server.

If you no longer wish to encrypt the data blocks, you can disable it.

Replicate data

If you are replicating data, you also have to define an encryption key on the replicated server to encrypt the replicated blocks. The key can be identical or different between the server and the replica.

During a restore

When an agent requests for a restore, decrypts the blocks prior to sending them to the agent requesting the restore. The blocks can be sent by HTTPS if needed.
