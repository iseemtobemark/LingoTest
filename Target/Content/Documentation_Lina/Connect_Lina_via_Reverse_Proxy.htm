Connect to via a Reverse Proxy

Administrators can set up a reverse proxy to access services over the Internet. Yet, agents require the reverse proxy public name or address to be able to connect, and this name is different from the corporate network server name.

Standalone configuration

In this configuration, you:

Must enter a line that specifies the corporate network gateway name or address and the server name or address on the corporate network.

The syntax to use for each network gateway is: <corporate_gateway_name_or_adress>=<private_aln_server_name_or_adress>

For instance: gateway=alnserver or 192.168.1.254=192.168.1.42

Can also enter a line that sets the reverse proxy public name or address as the current server name. The * wildcard is used to replace any gateway name, address or a part of it.

The syntax is: *=<reverse_proxy_public_name_or_adress>

For instance: *=myreverseproxy or *=128.30.52.37

Example. This is a configuration example of a client connecting to the server through a reverse proxy when the gateway is different from its corporate gateway:

gateway=alnserver

*=myreverseproxy

Multi-server configuration

In this configuration, you:

Must enter a line that specifies the corporate network gateway name or address and the master server name or address on the corporate network.

The syntax to use for each network gateway is: <corporate_gateway_name_ or _adress>=<private_aln_master_server_name_or_adress>

For instance: gateway=alnserver or 192.168.1.254=192.168.1.42

Can also enter a line to manage node access. The syntax to use for each node is:

<corporate_gateway_name_or_adress>|<node_name_or_address>=<private_aln_node_name_or_address>
