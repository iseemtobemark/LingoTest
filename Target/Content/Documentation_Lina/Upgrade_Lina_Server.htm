Upgrade the Server

Before upgrading the  Server, ensure that :

You have downloaded the binaries from the Atempo Client Portal: https://support.atempo.com.

The user launching the setup program has administrator or root rights.

If you intend to use the built-in mechanism to update your agents automatically (rather than using a tool such as GPO), you have set the Update URL parameter in the server prior to upgrading the server. If not, agents may become unreachable after the server update, at which point you are unable to push the URL to the agents. See Update Your Web Server Address for details on that parameter.

To upgrade the Server

Close all existing directories.

Launch the setup program corresponding to your operating system:

Windows Double-click the executable ATLXXX-Windows-X64.exe.

Linux Run ATLXXX_Linux-X64.bin. You need to ensure that the binary can be executed. If necessary, you can define the rights with the command: chmod +x.

Select an installation language.

Click Next.

Accept the License Agreement.

Upgrade, then click Next.

In the drop-down list, select the version to upgrade, then click Next. A summary indicates the installation parameters.

License File If you selected Atempo-License Manager during a server upgrade, you need to indicate whether you have the license file corresponding to . If you do, provide the location of the file on the server. If you do not, a temporary license file is automatically attributed.

Click Install, then click on Done.

If the upgrade fails and returns a message indicating the directory is locked, do the following:

Quit the upgrade.

Restart the ade.default.

Relaunch the upgrade in this procedure.
