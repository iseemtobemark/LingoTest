Create a Tenant

To create a tenant

Log into the with administration rights (superadmin).

Go to Server> Structure > Tenants to open the list of tenants.

Above the list of tenants, click + to create a tenant sheet.

Enter the tenant name.

(Optional) Add a comment below the tenant name.

(Optional) Define the tenant as a default tenant, using the toggle switch. You can enable this option to prefill the tenant field in the login window.

(Optional) Set up an automatic connection to the tenant when logging into .

This option automatically binds a machine name, IP address, or IP address set to a tenant without this tenant being the default one.

To specify how the connection should be bound:

Enter an IP address, a machine or a sub-network name.

Click Add this auto-connection.
If no address or name is specified, the auto-connection is taken into account.
If any auto-connection is set, the default domain is taken into account.

(Optional) These steps can be performed with new or existing tenants on a multi-server configuration only:

Select a dispatch policy that redirects agents to the nodes. By default, you can add an agent on alternating nodes (round robin), on a node with the least number of agents or with the most free space.

Select one or several nodes to associate with a tenant, using the toggle switch besides the tenant name. By default, a tenant can access all nodes but you can restrict the number of nodes that can be accessed. If you have nodes, you can add a tenant to one or several nodes.

Click Create to validate the creation of the tenant. If you want to edit the tenant properties, click on the tenant in the tenant list, edit the property of your choice, then click Save.

In the list of tenants, you can display or hide information regarding UUID, users auto-connections, nodes, comments or agents.
