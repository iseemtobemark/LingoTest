Main features

 is equipped with multiple features for backup protection of workstations and file servers (see Table):

features and descriptions

Feature

Description

Continuous Data Protection and Flexible Recovery Point Objective

offers automatic and near continuous protection of user data. There is no backup scheduling. Instead, new or modified data is detected in real time and automatically transferred to the server, without user intervention. The time-intervals between transfers of data to protect is defined by the Recovery Point Objective (RPO). The RPO can vary from one minute to one day.

Workstation recovery

The Bare Metal Recovery (herein called BMR) is a module used to recover a complete Windows agent (system and data). The BMR feature allows to backup and restore MS SQL Servers and requires setting a specific protection policy at backup time. This ensures that all necessary files are backed up and available if you need to perform a recovery of your machine.

Block Level Deduplication

implements incremental backup and deduplication technology at block level.

WAN Mode

reduces network traffic since already protected blocks are not re-sent to the server.

"Self Service" Restore of Agent Data

A lets the user restore files or folders at any time without administrator intervention. Data can be restored to its original place or to a new location.

Advanced Restore also enables to perform a:

cross-restore,

distant restore,

restore from an alternate server.

Versioning

keeps historical logs of protected versions, letting you choose the protection date or version of the file or directory to restore.

Support of Open Files

For continuous protection of user data, protects open files. Applications such as Microsoft Outlook and Microsoft Office do not need to be closed, their files are protected while in use.

Support of Windows EFS

EFS (Encrypting File Systems) encrypt data on a machine to ensure its confidentiality. When protects the data on an EFS system, it is protected and restored in its encrypted form.

Network Management

agents automatically connect to the server as soon as a network connection is available.

lets you fine-tune data protection by defining, among other things, the behavior to adopt for different types of network, or the compression algorithm for data transfer.

Centralized Handling of the Server, Free Configuration of the Agent

Administrators of the server can:

Oversee the entire IT environment of the company (file servers, workstations and laptops).

Apply common protection policies to groups of machines through a centralized .

Define enterprise-wide protection policies such as exclusions and inclusions, and rights for agents.

Depending on their rights, users can set up their own protection policies on the , as long as their customized policies do not conflict with the protection rules defined on the server.
