Configuring Protection Policies on the Server

Configuring protection policies on the server goes through the following stages:

Declaring Agents: it can be done automatically, but by default, it is in manual mode. In automatic mode, when an agent is installed on a machine, this machine is displayed in the of the server. See Declaring Agents for details on how to switch to automatic creation.

Applying a Protection Policy to the Agents. A protection policy contains:

A protection scheme to define the data to protect.

A strategy to define protection settings.

Predefined schemes and strategies are available on the server. You can use these as they are, modify them, or create new ones.
