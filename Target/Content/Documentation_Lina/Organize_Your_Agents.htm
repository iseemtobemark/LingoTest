Organize Your Agents

These topics describe the organization of agents.

To facilitate the administration of agents in the you can:

Divide the agents declared in the server into agent groups. You can create several groups but each of them must have its own name. A protection set to an agent group can be inherited by any agent added to that group.

You can create several views. Each one must have its own name. The agent groups created constitute a view. A view represents the highest level in the organization.

An agent group called No protection inheritance is automatically created in the . You cannot delete it but you can duplicate it. This group contains all the agents that have not yet been assigned to other groups. This group cannot be deleted.
