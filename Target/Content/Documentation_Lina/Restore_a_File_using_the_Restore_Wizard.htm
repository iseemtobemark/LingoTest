Restore a File Using the Restore Wizard

The lets you filter the list of files to restore by applying search criteria based on location, type, and time range.

Step 1: Launch the

Right-click the icon in the task bar and select .

Select Restore a file > Start.

Step 2: Select the restore source for the file

A list of available servers displays in the upper-right corner of the window. This list displays only if there are several restore sources available and if you are not in Advanced mode. Each item of the list contains:

An icon that specifies if the server is available or not.

The name of the restore source (primary and secondary).

The retention period for each server.

Select the server to use for the file restore.

By default, the primary server is selected as an available server. The switches on the secondary server if the primary server is unreachable.

Step 3: Locate the file to restore

In the Select Folder window, select Browse backup to locate the folder in which your file was saved. You can indicate either the complete path to your file, or enter a parent folder and choose to include sub-folders in the search.

Click OK. The folder path appears in the folder selection window.

Click Next.

(Optional) Step 4: Provide additional information

Type or select the extension from the Show existing file types list.

Click Next.

If you know the last modification of your file, specify the time and date range.

Click Next.

Step 5: Select the file to restore

Choose your file from the list.

Click Next.

Select the version of the file you want to restore.

Click Next.

Step 6: Define restore options and restore your file

Choose the restore location:

Turn off the toggle to restore the file to its original location.

 

Turn on the toggle to change the target directory of the restore.

If you choose a new location, click Mycomputer to add the target directory.

If the file to restore already exists at this location, select the restore behavior :

Overwrite The file is replaced by the restored file.

Skip The file is not restored.

Rename The file already present is renamed with the suffix .bak.

If needed, change the identity to a user that may have more privilege to restore data. If the chosen identity is linked to an admin group, the restore is done directly as administrator (NT Authority\system on Windows, root on Linux and macOS):

Impersonated restore Type the username and password of the user you want to restore.

System restore on Windows Type the username and password belonging to the local administrators group.

System restore Linux and macOS Type the username and password belonging to the admin, adm, sudo, wheel or root groups.

Click Check Identity.

Select Restore. To cancel the restore process, click Cancel.

Once the restore process is complete, you can view the details in the status overview (see Figure).

Click Finish to close the .

Successful file restore

The file size information in the is shown in decimal prefixes (1 MB is equal to 1000 KB) whereas file sizes in File systems are usually shown in binary prefixes (1 MB is equal to 1024 KB). As a result, you may notice a difference if you compare both sizes. The file in the seems bigger. However, they are actually of the same size.
