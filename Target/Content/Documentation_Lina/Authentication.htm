Authentication

Format

On Windows machines, authentication is done in one of these two formats:

netbios_domain_name\user

For instance: atempo\mike

user@fully_qualified_domain_name

For instance: mike@us.atempo.network

On Linux machines, the user name format can be:

user_name (for a local connexion only)

For instance: mike

user@fully_qualified_domain_name

For instance: mike@us.atempo.network

Supported accounts

The login you provide must be of an account allowed to log on to the local machine.

On Windows, the login must be interactive rather than a network login.

Two-factor authentication

A two-factor authentication method is available in the , Configurator and the Statistics interfaces. When a two-factor authentication method is enabled, users must enter their credentials, as well as an additional code.

To activate the two-factor authentication for a user

Download the authentication application of your choice (Authy, Google Authenticator, Microsoft Authenticator, etc.).

Click Server > Users.

In the drop-down list above the list of users, select a tenant for the agent.

To activate the two-factor authentication for an admin or a superadmin user, select All tenants.

Select a user to open an information page for this user. This page displays information related to:

The status of a user (enabled or disabled).

The user e-mail and the possibility for the administrator to change the password of a user, if necessary.

The user group selected for that user.

The double-factor authentication.

Select the 2-factor authentication tab.

Provide an authentication key to the application. A QR code and a 16-digit code are provided in the . They can be used as authentication keys. When an authentication key is provided, the application generates a 6-digit code every 30 seconds.

Add the account name and user password in the authentication application.

At the bottom of the page, enter the code provided by the authentication application, then click Enable the 2-factor authentication. A confirmation dialog is displayed when the code provided is correct. This dialog contains:

A button to disable the two-factor authentication for the user. A 6-digit code is required to disable the two-factor authentication.

A button to remove recovery codes. This button prevents the use of recovery codes when the two-factor authentication is enabled.

A button to generate new recovery codes. This button generates new recovery codes.

Ten recovery codes, which can be used once, are provided in the . These 8-digit codes display only when the two-factor authentication is enabled.

When the two-factor authentication is enabled for one or several users, the icon appears in the list of users.

To deactivate the two-factor authentication, remove recovery codes or generate new recovery codes, go to the agent details, then in the 2-factor Authentication tab.

To log in the , Configurator and the Statistics interfaces

Enter the user's credentials. An additional step is required to confirm the authentication. The user can:

Enter a security code provided every 30 seconds by the authentication application.

Enter a recovery code provided in the .

Contact an administrator to reset the user's account.

Enter a security code or a recovery code, then click Validate. To use the same code for 30 days, select Extend the code validity for 30 days.
