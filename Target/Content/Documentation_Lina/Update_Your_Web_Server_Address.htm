Update the Server Address used for Agents

The enables you to set the configuration parameters that only apply to new and existing agents.

To enable an automatic update of agents

Go to the Agents list.

Click on the button above the list of agents to configure the global agents parameters.

In the Declared agents tab, go to Update.

Enter the web server address that is used to perform automatic updates for the agents in a tenant.

The server address is retained unless you erase it or replace it by another server address.

Once you add an address, you can enable or disable automatic update checks and installations run by the server (Activate updates) (see Figure).

Activate automatic agent updates

This web server address must be defined for each tenant. Up to version 5.2, this feature requires a user with administrator privileges to update the agent (e.g. from a version 5.1.8 to 5.2).

When updating an agent 5.2 to a later version, the administrator is able to use the same URL mechanism to transparently push new agent versions. Users are informed to refresh the agent interface, no further action is required. The agent checks for new updates at launch and then every 24 hours.

You can make the contents of the directory from the product download available through the web server of your choice (either use an existing web server or install a new one).

The URL of this directory can be entered in the current field by copying the content of the directory ATLXXX.iso\Agent from the product download to the address http://<your_web_server_name/Lina_agent_update
