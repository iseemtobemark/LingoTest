Install the in Synology DSM 7

A DSM 7 Synology is installed using a Command Line Interface. The installation involves two stages:

Retrieval of the package (lina_5.3-XXXXX_X86_64.spk for Intel or lina_5.3-XXX_arm.spk for ARM, for example).

Installation of the package in the Synology administrator interface. Administrator rights are required to perform the installation.

If the is already installed on a Synology version DSM 7, you must uninstall it, then install the most recent version of the for Synology version DSM 7.

Install the

Step 1: Enable the SSH service

The SSH service must be enabled before installing the .

In the NAS control panel, click the Terminal & SNMP section.

In the Terminal tab, enable the SSH service and click Apply to confirm your choice.

Step 2: Assign access rights to the

Log in to the Synology login terminal as an administrator, using an SSH client. You can use a tool like PuTTY or a Linux terminal.

Create the HN directory, then modify the access rights for the by running the following commands:

sudo mkdir /usr/local/etc/HN sudo chmod -R 777 /usr/local/etc/HN sudo chmod 777 /usr/local/etc/rc.d/

Step 3: Install the package in Synology

Log in to the Synology web administration interface.

In the Package Center, click Installed in the left menu pane.

Click Manual Install. A file explorer displays.

Click Browse, then select the installation file. An error message indicating that the package is not verified by Synology displays.

Confirm the package installation.

Enter the server name.

Unselect the Run after installation option, then click Done. An error message indicating that the server has not been found may appear. This message is normal because the service is not started yet.

Click OK. The package is installed, but does not start automatically.

In the Synology terminal, run the following command:

sudo sed -i 's/package/root/g' /var/packages/lina/conf/privilege

Step 4: Start the

Start the via the Package Center of the Synology server administration interface.
