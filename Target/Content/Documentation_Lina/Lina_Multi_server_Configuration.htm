A multi-server configuration is composed of a main server (master server) that centralizes the management of several servers called nodes. This type of configuration can be used with only.

A multi-server configuration improves agents and server management. As a result:

You can use as many agents and servers as necessary. The number of agents is not limited anymore.

You can access information regarding agents and tenants, and monitor all your servers through a centralized administration interface.
