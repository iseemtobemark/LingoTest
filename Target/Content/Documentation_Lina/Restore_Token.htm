Restore Tokens

Tokens allow you to securely delegate cross-restore rights to agents for a specified period of time. A token is also required to perform a BMR restore or .

Access the List of Restore Tokens

In the , you can access the list of tokens via Restore > Restore tokens.

The interface displays the following information on each token:

Token code that must be provided for each cross-restore.

Date on which the token expires.

Type of action allowed when using the token (cross-restore or web restore).

Agent(s) affected to the token.

Number of times the token has been used.

Last use date.

Tenant within which the token has been created.

In the list of restore tokens, you can:

Add or delete a token.

Send a token by e-mail.

Copy the token code to the clipboard.

Refresh the list of tokens.

Export one or several token codes in CSV or HTML format.

Create a Restore Token

A user can create a restore token only if create rights are enabled in the user profile assigned to that user (see Figure). See Creating a User Profile for more details.

Create rights can be activated on the page in the user profile settings

To create a restore token

In the , select Restore > Restore tokens.

In the list of tenants above the list of restore tokens, select the tenant within which you want to create the token.

Click the button.

In the list of agents, select one or several agent(s) to associate with this token. If you want to use a unique token for existing and future agents, toggle the Universal token option.

Define an expiration date for the new token:

Unlimited: The token has no expiration date.

Limited with a date: The token has an expiration date and time.

Limited with a validity period: The token has a validity period.

Set the type of action that can be assigned to the token. A token enables a user to perform a cross-restore or a web restore.

Click Create to save the token. A token with a unique code is provided.

Send a Restore Token by E-mail

To send a token by e-mail to one or several user(s)

Select one or several token(s) in the list of restore tokens. Before sending the token(s), you can easily modify the e-mail address associated with the agent in the e-mail sending form.

Click the button.

Provide a user e-mail address, then click Send.
