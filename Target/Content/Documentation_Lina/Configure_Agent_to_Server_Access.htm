Configure Advanced Agent to Server Access

The ports and proxy (if any) information required for the agent to access the server is integrated with the server name definition in the CfgServerHost file in the form:<tenant_name>/<agent_identifier>:<node_name>@<server_host>:<port>::<proxy_host>:<proxy_port>:<proxy_user>:<proxy_passwd>

If a parameter is not specified, its default value is used:

The most basic definition is:

<server_host>

The Agent Identifier is the name of the as displayed in the . The Agent Identifier can also be modified in the Agent view of the . See To display the general agents view. By default, the Agent Identifier is identical to the computer name. However, you can declare a different Agent Identifier for your agent during the installation process:

Standalone configuration: This command is used to add an agent to a tenant: <tenant_name>/<agent_identifier>@<server_host> where server host is a standalone server.

Multi-server configuration:

This command is used to add an agent to a node via a dispatch policy. <tenant_name>/<agent_identifier>@<server_host> where server host is a master server.

This command is used to force an agent to a specific node. <tenant_name>/<agent_identifier>:<node_name>@<server_host> where server host is the master server and node name is the access string in the . To find the access string, see Managing Nodes.

This command is used to add an agent to a tenant:

<tenant_name>/<agent_identifier>@<server_host>

Server host:

To redefine ports, use:

<server_host>:<port>

To redefine the proxy and use default value for the port numbers, use:

<server_host>:::<proxy_host>:<proxy_port>

To redefine the proxy and proxy authentication and use default ports, use:

<server_host>:::<proxy_host>:<proxy_port>:<proxy_user>:<proxy_passwd>

For instance, if you are deploying on a Windows agent, the command is in the form:

setup.exe /vSERVERNAME=<server_host>:::<proxy_host>:<proxy_port>:<proxy_user>:<proxy_passwd>

If you are using the graphical interface when you specify the server name, use:

<server_host>:<port>::<proxy_host>:<proxy_port>:<proxy_user>:<proxy_passwd>
