Create and Assign Rights to a Local User

When installing , a local administrator is created by default with the following identity:

Username: superadmin.

Password: superadmin.

It is strongly recommended that you modify the password of the superadmin user as soon as possible. To reset the password of an existing user, go to step 9 of the following procedure.

To add a user

Log into the with administrator privileges.

Select Server > Users.

Assign a tenant to the user using the drop-down list in the top bar of the .

To create a regular user or an administrator, select a specific tenant.

To create a superadmin user, select All tenants.

In the Users view, click the button.

The user parameters interface appears.

Type the local user name.

You can change the user name after the user creation. For an LDAP user, the user must be in the form: user@domain.

(Optional) Add a comment.

(Optional) Select Local user as the type of user.

(Optional) Enable or disable the user. By default, users are enabled. A disabled user cannot log into the .

Provide a password for the local user.

When a user is created, the password field is replaced by an option. To reset the user password, toggle the Change the password, then enter a new password.

(Optional) Provide the e-mail address of the local user.

The email address can also be modified on the . For more details, see the .

(Optional) Select a user group to grant rights to the user. A profile is associated with a user group and therefore grants rights to it. If no group is selected, new users are added to the Default User Group. See Creating a User Profile and Creating a User Group for more details.

Click Create.

If you want to edit information of a local user, click the user name in the list of users, then edit the user details and click Save.
