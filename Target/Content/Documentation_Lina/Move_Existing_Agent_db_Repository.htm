Move an existing Agent Database Repository

Moving an agent database repository does not remove the source repository. Once agent database repositories have been moved, and no new repository is added, continues using the old repository for the incoming new agent. See Adding or Removing an Agent Database Repository for details.

To move an existing repository contents:

Stop the Deduplication Engine service (by default ADE.default).

Launch the command ade_server -L "src=<sources>,dest=<destination>,set_as_default=[0|1],vol_max=<size in bytes>"

Where:

<sources> is the list of repositories to be moved. If not provided, move all existing repositories.

<destination> is the list of destination repositories.

<set_as_default> is 0 if the source repositories remain to be used as agent database repositories. Otherwise, set the destination as the new agent database repository.

<vol_max> keeps moving agent database repositories until the limit is reached (optional).

Restart the Deduplication Engine service.
