Deploy a Windows Agent Remotely

Windows agents may be deployed remotely, using either an Active Directory Group Policy or a deployment tool such as WAPT, Microsoft SCCM or equivalent. Only one agent is needed per machine, even in an RDS/TSE environment.

Before deploying agents remotely, the user performing the installation on a machine must have administrator rights on that machine.

Step 1: Extract the MSI file

Run the setup.exe /extractcab command in the command prompt. This command will create a SupportFiles folder containing the ALN.msi file to extract.

Step 2: Launch the MSI package

Launch the MSI package in silent mode, using one of the following commands:

ALN.msi /qn SERVERNAME=your-lina-server

msiexec /qn /i ALN.msi SERVERNAME=server

See Installing [%=ALN.aln%] Windsows Agents in Silent Mode for detailed information on mandatory or optional parameters.
