Create Agents Manually

In the , you can add agents manually to the list of agents to protect. Once they are declared in the list, you can apply a protection scheme and strategy to each one individually.

Open the

Select Agents to open the Agents view.

In the list of tenants, specify the tenant to associate with the agent.

Click the button.

The new agent details window appears. (see Figure).

New agent parameters

Enter the name of the that you want to declare or select an agent from the list of agents that have been installed but not yet declared.

When declaring a macOS agent named by default "user’s Mac", we recommend to select the agent from the list rather than typing its name. If you do type it, note that the apostrophe in "user’s" is a typographic apostrophe (unicode 2019, UTF-8 E2 80 99) and not the apostrophe from the keyboard (0x27).

Select an agent group.

By default, agents inherit the protection and strategy set for their agent group. Agents can have their own protection and strategy if they belong to the default agent group (No protection inheritance).

You can also add an agent to an agent group in the list of agent groups.

Select the protection scheme to apply to your agent.

You can also create a new protection by selecting the + button or view information on that protection by clicking the pen icon.

Select the strategy to apply to your agent.

You can also create a new strategy by selecting the + button or view information on that strategy by clicking the pencil icon. Your agent inherits the strategy selected for his agent group.

Select a status for the agent.

Agent statuses distinguish active agents from inactive agents. They also determine which operations can be run on the agent. For more information, see (Block statuses) see Table. Available statuses are:

Keep all versions (normal state).

In vacation.

Paused.

Manually disabled.

Archived.

Keep only last version.

Select an update policy. You can use the default update policy or create an update policy. To create an update policy:

Click Use a specific update policy.

Enable updates.

Add an URL of update.

Enable the Force update as soon as possible option.

(Optional) Select a tag to apply on the agent. A tag can be used to highlight a characteristic common to several agents (a location, for instance).

To select an existing tag, click in the tag field and enter a tag name. Click on the tag automatically suggested to apply this tag on the agent.

To create a tag, click in the tag field and enter a tag name. Pick a tag color, then click the + button.

The tag is automatically applied on the agent.

Click Create to finish declaring the agent.

The agent is added to the list of Agents, in the No protection inheritance group.
