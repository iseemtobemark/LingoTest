Monitor an Agent

Backup configurations made on the can be monitored by an administrator in the agent details.

To display agent details

In the left menu of the , click Agents.

In the list of agents, click on an agent.

The agent details display.

Consult Information on the configuration

Select the Information section to consult general information regarding the workstation on which an agent is installed (Operating System or agent email, for instance). This information is retrieved by the server.

Monitor Backup Configurations Edited on an Agent

Select the Local configurationsection to consult information on the local backup configuration of an agent (maximum acceptable data loss, for instance). This information is sent from the .

Monitor an Agent Activity

When a problem occurs on an agent, an administrator can consult the agent logs on the . A list of events is automatically retrieved by the server. That list:

Can be consulted in the server events and is sorted per severity, thread, source, date and description. This list retrieves information on all agents of one or all tenants.

To sort events, go to Server > Events, then select a type of severity in the filters.

Can be consulted in the agent details and is sorted per severity, description, source and date.

You can sort events, using the list of events sent by a specific agent or retrieve specific events upon demand.

You can:

Limit the list of events collected from the agent, using the filter above the list.

Retrieve a detailed list of events for a specific type of log, using log configurations.

To filter an agent activity

Select the Local events section.

Enter the severity, thread, source, date and description in the filter field, then click on the magnifying glass.

To retrieve a detailed list of events, using a log configuration

Contact the support team for more details on the log category to use.

Select the Local events section in the agent details.

Enter the log category given by the Support team.

Select a severity type, then click the + button.

A button named under the log category displays above the filter field.

Click Save.

The list of events will be updated when new details are transferred from the agent to the server.
