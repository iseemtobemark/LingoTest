Assign an E-mail Address to the Agent

This e-mail address can:

Be provided in the for each agent or several agents.

Be provided in the agent interface. To provide an e-mail address in the agent interface, see Providing an E-Mail Address to Identify the Agent.

Be used to send e-mail notifications.

Be used to access the interface.

Assign an E-mail Address in the Agent Details

To manually assign an e-mail address to an agent in the :

In the left menu of the , click Agents.

In the list of agents, click on an agent.

The agent details window appear.

Go to Information > User, then enter an agent e-mail.

If a user e-mail is added in the , this e-mail is not editable in the agent interface (Identity tile in the agent settings).

E-mail field in the User area

Click Save.

Go to Server > Users.

Click on a user.

The user details window appears.

In the E-mail section, enter an e-mail address to identify the agent (see Figure). You can provide multiple addresses by using a colon to separate them.

E-mail text field in the User area

Click Save.

Assign an E-mail Address, using the Active Directory

To automatically assign several e-mail addresses to several agents at a time, using the Active Directory:

Go to the list of agents.

In the list of agents, click the icon above the list of agents to access the agents settings.

In the agents settings, go to Declared agents > E-mails update.

Ensure the LDAP is configured.

You can display the LDAP configuration. If no LDAP is configured, see Configure LDAP.

(Optional) Overwrite existing e-mails to update the whole list of user e-mails. If you do not enable this option, no existing user e-mail is updated.

(Optional) Click on the Test button to preview the e-mails update.

Click on Update agents e-mail now to apply all changes.
