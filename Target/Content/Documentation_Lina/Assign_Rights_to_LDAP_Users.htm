Assign Rights to LDAP Users and groups

Rights can be assigned to new LDAP users only. You must create an LDAP user before assigning rights to that user.

To assign rights to LDAP users and groups

Log into the with administrator privileges.

Select Server > Users.

Assign a tenant to the user, using the drop-down list in the top bar of the .

You can associate a user with all the tenants or a specific tenant.

In the Users view, click the button. The user parameters interface appears on the right.

Type the local user name. You can change the user name after the user creation. For an LDAP user, the user must be in the form: user@domain.

(Optional) Add a comment.

Select the type of user (LDAP user or LDAP group).

If no LDAP is configured on your server, you may see a reminder. To configure the LDAP, just click the Resolve button below the reminder.

Enable the Browse LDAP option.

Enter the ID and password of a user allowed to connect to the LDAP server to retrieve the list of groups.

If needed, enter a filter that will be applied to the group names to reduce the length of the group list.

Click the Get LDAP users button for an LDAP user or Get LDAP groups for an LDAP group.

(Optional) Enable or disable the user. By default, users are enabled. A disabled user cannot log into the .

(Optional) Provide the e-mail address of the user.

The email address can also be modified on the . For more details, see the .

(Optional) Select a user group to grant rights to the user.

A profile is associated with a user group and therefore grants rights to it. If no group is selected, new users are added to the Default User Group. See Creating a User Profile and Creating a User Group for more details.

Click Create.

If you want to edit information for an LDAP user or group, click the user name or LDAP group in the list of users, then edit the user details and click Save.
