Retention

The retention is the period, in days, during which a file remains restorable after it has been:

Deleted from the agent

Excluded from the protection policies

Replaced by a new version of the same file

Until a protected file falls into one of these three situations, the retention countdown does not start and the file remains restorable at any time. When a file can no longer be restored, it is picked up by the background tasks and removed. The removal triggers the de-referencing of the datablocks so the ADE subsystem compactor eventually could free up part or all of the space in the block repository.
