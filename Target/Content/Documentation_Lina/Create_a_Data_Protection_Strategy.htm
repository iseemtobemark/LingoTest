Create a Data Protection Strategy

Step 1: Create a strategy

Open the .

Select Backup > Strategies.

Select a tenant in the Current tenant list at the top of the page.

Click .

The New strategy interface is displayed.

Enter the name of the strategy you want to create.

(Optional) Step 2: Configure the server section

Set a value for all or part of these parameters:

Retention period Define the number of days for which data remain restorable once the retention countdown starts. The default retention period for a new strategy is 30 days. This new retention period does not affect existing strategies.

See Retention for details.

Replication Select the destination to which data will be replicated. Enable the Limit the retention option to define the number of days for which replicated data remain restorable once the retention countdown starts.

This parameter only appears if you have created at least one replication destination.

Quotas Set a limit to the protected volume

To define a limit, specify a value and unit for quotas. This limit is applied to all agents using this strategy. See Quotas to Manage Protected Data Volumes for details.

(Optional) Step 3: Configure the agent section

Set a value for all or part of these parameters:

Never lose more than Define the maximum time lapse from one minute to one day, between two protection sessions, indicating the maximum acceptable data loss if data is restored between two sessions. If the protection settings are modified on the server or the agent, the protection relaunches immediately, without waiting for the time defined in the RPO.

Raise an alert if an agent has not been protected Specify the time threshold to apply between the moment an agent is no longer being protected and the moment an alert is generated in the .You can specify this threshold in days, weeks, or months.

Agent Rights Grant users the right to modify their own settings on agents (such as modifying the bandwidth or limiting agent browse to the user home directory (see Limit the Agent Browse to the User Home Directory on ).

(Optional) Step 4: Configure the network section

Select a compression algorithm to transfer your protected data over a network:

Raw is an uncompressed format.

ZLIB offers the best compromise between speed and compression rate. Recommended for LAN connections.

LZMA supplies better compression, but requires more processing power. Recommended for slow connections such as WAN.

Brotli is a compression algorithm offering a decompression speed comparable to the deflate algorithm, and a compression ratio close to LZMA.

Zstd is a compression algorithm offering a compression ratio close to Brotli, as well as as a low CPU usage.

Enable all or part of these parameters, then set a value for each parameter enabled:

Limit throughput Define the amount of bandwidth to enable for the transfer of data between the server and the agent. Specify a value in KB/s.

Restrict network interfaces Set conditions on the network interface as well as the gateway interface.

If it is selected, you must specify a network interface and gateway for the agents to use to communicate with the server. If it is not selected, all agents remain accessible to the server over the network, whichever gateway and network interface are used.

Use WAN mode Enter the name of a gateway on which the WAN mode should not be activated.

You can use the character * in the definition of the gateway name, and enter several gateways separated by a comma. The WAN mode is a network inter-agent deduplication optimization. It makes it easier to perform backups in a slow network environment, ensuring the agent does not send blocks already stored on the server.

Enable or disable encryption when data is transferred over any network gateway.

If you select this option, you can enter the name of the gateway on which the encryption of data must not be enabled. You can enter several gateways separated by a comma. Only the superadmin can enable or disable this option.

Click Create to finish declaring the strategy.

The strategy appears in the list of strategies. To modify an existing policy, select it from the list, and modify the server configuration and the agent configuration.
