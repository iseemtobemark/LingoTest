Restore a File Using the Browse View

The agent interface is equipped with a Browse mode that allows you to restore multiple files at once. If you want to learn more about the Browse interface before starting the restore, see About the Browse View.

Right-click the icon in the task bar and select .

In the Restore menu, select Browse.

A list of available servers displays in the upper-right corner of the window. This list displays only if there are several restore sources available and if you are not in Advanced mode. Each item of the list contains:

An icon that specifies if the server is available or not.

The name of the restore source (primary and secondary).

The retention period for each server.

Select the server to use for the file restore.

By default, the primary server is selected as an available server. The switches on the secondary server if the primary server is unreachable.

Choose the date from which you want to restore your file(s) with the Time Navigator slider or the calendar.

displays your agent file tree at that date.

From the file tree, check the box corresponding to the file to restore (see Figure).

Restorable files using the Browse view

Select the version you want to restore:

To restore the latest backed up version of the file, select Restore. To cancel the restore process, click Cancel.

 

To restore a different backed up version, select the icon of your file to see the available versions and select the icon to restore that version.

Choose the restore location. By default, the file is restored to its original location, but you can switch the toggle on to restore the file to another location.

If you choose a new location, click Browse computer to add the target directory.

If the file to restore already exists at this location, select the restore behavior:

Overwrite The file is replaced by the restored file.

Skip The file is not restored.

Rename The file already present is renamed with the suffix .bak.

If needed, change the identity to a user that may have more privilege to restore data. If the chosen identity is linked to an admin group, the restore is done directly as administrator (NT Authority\system on Windows, root on Linux and macOS):

Impersonated restore Type the username and password of the user you want to restore.

System restore on Windows Type the username and password belonging to the local administrators group, then click Check identity.

System restore on Linux and macOS Type the username and password belonging to the admin, adm, sudo, wheel or root groups.

Click Check Identity.

Select Restore. To cancel the restore process, click Cancel.

Once the restore process is complete, you can view the details in the status overview.

Click Finish to close the restore process.
