Agents in a Multi-Server Configuration

The workstation on which is installed must be associated with a node. You can:

Define an automatic allocation policy for agents on the nodes associated with a tenant. For more details, see Create a Tenant.

Associate an agent with a node while creating the agent on the .For more details, see Create [%=ALN.aln%] Agents Manually.

Specify the node to associate with the agent, using the accessinfo. For more details on the CfgServerHost, see Configure Advanced Agent to Server Access.

If the master server is unavailable for any reason, the agents continue to be protected and data can be restored. The master server is not required for backup or restore.

If a node is offline, agents associated with that node cannot be backed up or restored.
