Tenants

A tenant is a part of the server that separates a set of agents with their own properties and identity.

You can create an architecture with multiple tenants, that allows to manage the tenants with the same server by maintaining a perfect individuality of each tenant. You can also copy strategies, protections configurations, protection rules and zones, and file categories between tenants on the same server or on different servers.

To open the list of tenants, go to Server > Structure > Tenants.

In the list of tenants, you can:

Add or delete a tenant.

Select a tenant as a default tenant.

Refresh, filter or export the list of tenants in CSV or HTML format.

Display or hide specific information for all tenants, such as the UUID, users auto-connection, the nodes that contain each tenant, comments or the number of agents contained in each tenant.
