Hyperstream Replication Configuration

To activate Hyperstream replication, you must delete your global replication destinations. As long as at least one global replication destination is present, Hyperstream replication is disabled.

To use Hyperstream replication, you must:

Add a replication destination.

Configure a media pool.

Step 1: Add a destination

Connect to the with administration rights.

Select HyperStream > Replication.

Click Add. The New Replication Destination window is displayed.

Enter the parameters as described in this table:

Name Name of the new destination, as displayed in the .

Status Selecting the Status check box enables the replication destination.

Host Name Host name of the target server to which the data is replicated.

Port Destination port on the target server. The default value is 23232.

Select Validate the creation.

(Optional) To modify a destination:

If you change the information related to a replication destination, delete manually every former stream.

Double click on the replication destination you want to modify. The Destination Details window opens.

Modify the parameters.

Step 2: Configure a media pool

Perform a backup if you do not have a valid Hyperstream backup, and wait until it is completed.

Select HyperStream > Replication. A new pool has been created based on the name of the related media pool. Streams are grouped together by media pool. A special pool called empty is used to store every stream whose media pool is unknown.

Select the newly added media pool.

Click Modify.

Select a replication destination from list. The replication starts as soon as you enable the media pool.
