Scenario 3 - Restore of a Damaged Mirror

This diagram (see Figure) illustrates when restoring a damaged mirror.

Scenario 3

Prerequisites

Both s must have the same hardware, same operating system and same HyperStream installation path

Do not run any backups during the restore

Hypothesis

Mirrored configuration HyperStream_A to and from HyperStream_B

HyperStream_B is lost

Procedure

Stop HyperStream_A.

If available, from HyperStream_B, secure a copy of the conf/config.xml file.

Overwrite HyperStream_B with a full copy of HyperStream_A (installation path + all repositories + all streams).

Replace the file config.xml in the Conf directory of HyperStream_B with the one copied in step 2. If the file is not available, you will find a copy of config.xml in the first repository.

Start HyperStream_B.

Start HyperStream_A.

Verification

Use the resync_hss command to display the number of streams to send or get. If the mirrors are synchronized, there should be no stream to send or get.
